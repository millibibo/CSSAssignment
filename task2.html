FRONTEND ENGINEERING ASSIGNMENT 3: CSS

2) Container Queries and Grid

Hier ist eine Übersichtsseite für Rezepte zu sehen. Ein besonderes Gericht (z.B. Gericht des Tages) wird oben in groß angezeigt. Darunter folgen weitere Gericht, immer vier in einer Reihe. Alle haben ein Bild, einen Titel und einen Button, mit dem man zum richtigen Rezept kommt. Das Ganze ist in einem Gridlayout angeordnet, wobei das erste Gericht über alle vier Spalten angezeigt wird. Außerdem ist das oberste Gericht mit ContainerQueries gestaltet. Ist genug Platz auf dem Bildschirm, so wird das Bild links über zwei Spalten angezeigt und rechts daneben der Titel und noch ein Text. Außerdem ist der Button größer. Ist weniger Platz vorhanden, so nimmt auch das Rezept den Style von den anderen Elementen an.

ContainerQueries sind noch nicht mit allen Browsern zu 100% kompatibel. Firefox zum Beispiel hat sie laut Can I Use noch gar nicht implementiert. Dort ist auch Baseline nur mit Limited availability angegeben. Auch auf dem Interop Dashboard gibt es keine Informationen zu ContainerQueries, auch auf dem 2024 Dashboard stehen sie nicht. Damit sollten Container Queries noch nicht in Produktion verwendet werden. Eine Falllback Lösung wäre auch wieder mehrere Styles zu definieren und das große Layout dann über die Media screen Queries zu steuern.
